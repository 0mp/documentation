\label{insn:ldga}
\subsection*{LDGA: load a DTrace built-in variable}

\subsubsection*{Format}

\textrm{LDGA \%rd, var, ndx}

\begin{center}
\begin{bytefield}[endianness=big,bitformatting=\scriptsize]{32}
\bitheader{0,7,8,15,16,23,24,31}\\
\bitbox{8}{op}
\bitbox{8}{var}
\bitbox{8}{ndx}
\bitbox{8}{rd}
\end{bytefield}
\end{center}

\subsubsection*{Description}

The \instruction{ldga} instruction looks up the value of a DTrace
built-in variable based on the value in \registerop{var} with an
optional array index in \registerop{ndx}. \\

Contrary to  \instruction{ldgs}, the variable identifier is 8 bits long, and
the other 8 bits are used to act as an array index.

\subsubsection*{Pseudocode}

\begin{verbatim}
%rd = var[ndx]
\end{verbatim}

\subsubsection*{Constraints}

\subsubsection*{Failure modes}

This instruction has no run-time failure modes beyond its constraints.
